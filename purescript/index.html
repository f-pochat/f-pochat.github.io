PureScript

Introduction

PureScript is a strongly typed and functional programming language. It was initially developed by Phil Freeman in 2013. He started to work on it because he was not satisfied with the attempts of compiling Haskell to JavaScript.

PureScript can be mainly compared to Haskell. Which is one of the most popular functional programming languages. However, PureScript does not support all things Haskell does. For example, generalized algebraic data types or type families.

Benefits

We can find several benefits of choosing PureScript for a project:

Compile to readable JavaScript and reuse existing JavaScript code

The fact that PureScript can compile compile to readable JavaScript brings many pros. First, accessing to all the libraries and uses of JavaScript. For example, the most used front-end library for web development - React - can be used completely in PureScript. Then, the introduction of Foreign Function Interface makes PureScript operate with JavaScript and viceversa.

Strong type system

PureScript has a strong and static type system, which helps to catch bugs and errors at compile-time rather than runtime. This can lead to more reliable and maintainable code.

Active community

PureScript has a strong and active community of developers who contribute to the language and its ecosystem of libraries and tools. This can make it easier to get help and support when working with PureScript.

Concurrency

PureScript's support for concurrent programming can make it easier to write code that can be parallelized and run concurrently on multiple CPU cores. This can lead to improved performance in certain types of applications.

Build real world applications using functional techniques

PureScript is a purely functional programming language, which means that it is based on the idea of expressing computations as the evaluation of pure functions. This can make it easier to reason about and debug code, as well as include all the benefits that functional programming can provide.

Disadvantages

However, we can find some disadvantages of choosing PureScript for your next project

Functional programming can be difficult at first

While PureScript is relatively easy to learn for developers who are already familiar with functional programming concepts, it can have a steep learning curve for those who are new to functional programming.

PureScript has not been widely adopted

PureScript has a smaller community of developers compared to some other popular programming languages, which can make it harder to find resources and support. Also, it has a smaller ecosystem of libraries and tools compared to more established languages like JavaScript or Python. This can make it harder to find libraries that provide the functionality you need.

Performance

While PureScript can be compiled to efficient JavaScript code, it may not be as fast as some other languages that are designed specifically for performance, such as C or C++.

PureScript vs Haskell

Strict vs Lazy

Haskell has lazy evaluation, this means that functions and expression are not evaluated until needed. While PureScript has strict evaluation, where expressions are evaluated as soon there are bounded to variables. However, in PureScript there are ways to get laziness.

Language features and ecosystem

Haskell, as a more mature language, contains more language features. For example, generalized algebraic data types or type families are not supported in PureScript. Regarding their ecosystems, if we consider only PureScript and Haskell, the second one contains far more packages and tools. Nevertheless, with the FFI (Foreign function interface) in PureScript we can have access to the entire JavaScript ecosystem.

Community

PureScript is a relatively new language compared to Haskell. This gives Haskell the advantage of been a more mature language, with a bigger community that supports it. Big tech companies reported that they use Haskell. For example Facebook use it in Sigma, its system to detect spam and malware

PureScript vs JavaScript

The are also some key differences between JavaScript and PureScript:

Type System

JavaScript is a dynamically typed language, which means that types are determined at runtime. PureScript is a statically typed language, which means that types are checked at compile-time. This can make PureScript more reliable and easier to debug, but also more complex to write.

Syntax

JavaScript and PureScript have similar syntax, but there are some differences. For example, PureScript uses infix notation for function calls - as in Haskell - , whereas JavaScript uses dot notation.

Functionality

PureScript is a functional programming language, which means that it is based on the idea of expressing computations as the evaluation of pure functions. This can make it easier to reason about and debug code, but may also require a different way of thinking for developers who are used to imperative programming. JavaScript, on the other hand, is a multi-paradigm language that supports both functional and imperative programming styles.

Recomendations

There are a few recommendations that can be followed before starting with PureScript:

Start with the documentation

PureScript has extensive documentation available at https://purescript.org, including tutorials, a language reference, and a library reference.

Use a build tool

PureScript code is usually compiled using a build tool such as Pulp or Spago. These tools can help you manage dependencies, run tests, and build your project.

Practice good code organization

PureScript code can be organized into modules, which can help you structure your code and avoid naming conflicts.

Use type annotations

PureScript is a strongly typed language, which means that you should annotate your functions with their types. This can help you catch mistakes early and make your code easier to understand.

Just as in Haskell, type annotations can be added before each function declaration.

add :: Int -> Int -> Int
add x y = x + y

Use a package manager

PureScript uses the package manager Pursuit (https://pursuit.purescript.org) to manage libraries and dependencies. This can make it easier to find and use libraries in your projects. There are libraries for UI development, data storage, network communication, and more.
