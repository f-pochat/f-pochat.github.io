Custom Types

Struct

type Entry = {
	firstName :: String,
	lastName :: String,
}

Synonym

type AddressBook = List Entry

Functions

Giving the function a parameter type and return type is optional but is good practice since it acts as a form of documentation.

showEntry :: Entry -> String

The character <> is used for appending.

showEntry entry = entry.lastName <> ", " <> entry.firstName

Example

findEntry :: String -> String -> AddressBook -> Maybe Entry
findEntry firstName lastName book = head (filter filterEntry book)
	where
		filterEntry :: Entry -> Boolean
		filterEntry entry = entry.firstName == firstName && entry.lastName == lastName

Two functions are imported from Data.List.

filter :: (Entry -> Boolean) -> AddressBook -> AddressBook
head :: AddressBook -> Maybe Entry

So, basically, we are finding the first occurence in the address book.

Anonymous Functions

\user -> user.name

... filter (\id -> id == "1") ["1", "1", "3", "5"]

Function Composition

We use <<< or >>> for composition. The first is "backwards composition", and the second is "forwards composition". So, in the last example we could write

findEntry firstName lastName = head <<< filter filterEntry
  where
    ...

and equally would be

findEntry firstName lastName = filter filterEntry >>> head
